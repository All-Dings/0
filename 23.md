#### Blog-Chain-Time 2023.07.07-14:01 in Tübingen <a id="10401"/>
###### Topics: TeX, Dings-Tool, History

For my [last](#10102) Blog-Entry I introduced into the [Dings-Project](300000006.md) the [Markup-Language](9300000.md) [TeX](9300009.md), which especially allows to render Mathematical-Symbols in a quite beatiful Way.

TeX has been invented between the 1970th and 80th by the Computer-Legend [Donald_Knuth](70000040.md) and then it was used in the early 80th by [Leslie_Lamport](70000041.md) to implement his [Lamport-TeX](9300010.md) or shortly LaTeX. The latter is the most widespread today and also I have used it extensively for [my](1971092002.md) Studien-Arbeit in 1995/96.

Here is a short TeX Code-Fragment for an [Equation](13000026.md), ...

```
$$ H = \sqrt[12]{2} \approx 1,0595 $$
```

... which is then rendered as:

$$ H = \sqrt[12]{2} \approx 1,0595 $$

A single "`$`" starts and ends the Math-Mode for TeX and a double "`$$`" is used for centered unlabled Math-Equations. Fortunately the [Pandoc-Tool](9000099.md) allows us to use these Formulas also in [Markdown](31.md)-Files.

From the different Pandoc-Options for translating [TeX](9300009.md) into [Html](9300001.md) I use the `--mathml` Option that, as the Name indicates, produces [MathMl](9300008.md) as Output.

The upper Example is translated then into the following Mathematical-Markup-Language:

```html
<math display="block" xmlns="http://www.w3.org/1998/Math/MathML">
 <semantics>
  <mrow>
   <mi>H</mi><mo>=</mo>
   <mroot>
    <mn>2</mn>
    <mn>12</mn>
   </mroot>
   <mo>=</mo>
   <mn>1</mn><mo>,</mo><mn>0595</mn>
  </mrow>
  <annotation encoding="application/x-tex"> H = \sqrt[12]{2} = 1,0595 </annotation>
  </semantics>
</math>
```

As we already saw, the rendered Result already looks quite nice, but unfurtunately there is no Support for Labeling and Referencing the Equations. Therefore I extend the Dings-Tool once more and introduce a new Syntax for appending a Label and an Anchor:

```latex
$$ Latex-Expression $${#LABEL:ANCHOR}
```


For a nice Formatting I use a [Html-Table](2000215.md) with three Columns, putting the Equation in the middle and the Anchor to the right. I also add new Css-Classes, like "TeX-Table" or "TeX-Tr" to the [Dings-Css-File](300000014.md).

The following Example ...

```latex
$$ H = \sqrt[12]{2} \approx 1,0595 $${#0:10480}
```

... is then translated into this Html-Code, ...

```html
<table class="TeX-Table">
  <tr class="TeX-Tr">
   <td class="TeX-Td-Left">
     <a id="10480"/>
   </td>
   <td class="TeX-Td-Equation">
     <math display="block" xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>H</mi><mo>=</mo><mroot><mn>2</mn><mn>12</mn></mroot><mo>≈</mo><mn>1</mn><mo>,</mo><mn>0595</mn></mrow><annotation encoding="application/x-tex"> H = \sqrt[12]{2} \approx 1,0595 </annotation></semantics></math>
   <td class="TeX-Td-Right">(0)</td>
  </tr>
</table>
```

... that is finally rendered as:

$$ H = \sqrt[12]{2} \approx 1,0595 $${#0:10480}

## Day-101: Heimat-Programm & Pythagorean-Comma again <a id="10100"/>

#### Blog-Chain-Time 2023.07.04-23:00 in Tübingen <a id="10102"/>
###### Topics: Music, Mathematics
 
Because after on [our](#9101) Journey to Freiburg, I was not 100% sure, if my Thoughts about the [Pythagorean-Comma](90000013.md) were correct, I want to try a [Mathematics](13000000.md)-[Proof](600022.md) now.

According to [Wikipedia](190000006.md) [[0]](#10120) the Pythagorean-Comma is 23,46 [Cent](90000025.md) and in the Equal-Temperament one Cent is defined as one hundredth of a [Half-Tone](90000024.md).

Since in this System one Half-Tone is exactly one twelfth of an [Octave](90000015.md) [[1]](#10121), we have to apply to be Base-Frequency twelve Times the [Multiplication](13000021.md) of the Frequency-Ratio of **H** in order to reach the Octave **O**:

$$\operatorname{Base-Frequency} * H * H * H * H * H * H * H * H * H * H * H * H = O$$

Because we choose $Base$-$Frequency=1$ and we already know that the Frequency-Ratio of the Octave is 2:1 or simply 2, we can write this as:

$$H * H * H * H * H * H * H * H * H * H * H * H = 2$$

Using the [Exponentiation](13000022.md) we can formulate this also much shorter as:

$$H^12 = 2$$

In order to calculate **H**, we have to apply Exponentiation with $\frac{1}{12}$ on both Sides:

$$(H^{12})^{\frac{1}{12}} = 2^{\frac{1}{12}}$$

After shortening the [Fraction](13000007.md) $12*\frac{1}{12} = 1$ and replacing $2^{\frac{1}{12}}$ with the more common [Root](13000023.md)-Writing $\sqrt[12]{2}$, we get the following Frequency-Ratio for our Half-Tone:

$$H = \sqrt[12]{2} \approx 1,0595$$

As mentioned at the Beginning, according to [Wikipedia](190000006.md) the Pythagorean-Comma is 23,46 [Cent](90000025.md) and in the Equal-Temperament one Cent is defined as one hundredth of a [Half-Tone](90000024.md), so one could naively assume that we now can calculate **Comma-Expected** as:

$$\operatorname{Comma-Expected} = 23,46\, Cent * \frac{1}{100}$$

Unfortunately it is not such simple, because in our Case we don't have a linear System. Therefore the Calculation of the Frequence-Ratio for the Half-Tone was the wrong way, sorry by bad. Instead the Frequency-Ratio **FR** of **CT** Cents can be calculated [[2]](#10122) with:

$$FR = 2^{\frac{CT}{1200\, Cent}}$$

The Reason for the 1200 Cent is the Octave again, that is exactly 12 Have-Tones, or $12 * 100\, Cent = 1200\, Cent$, so $2^{\frac{1200\, Cent}{1200\, Cent}} = 2^1 = 2$, which is - as we already know - the Frequency-Ratio 2:1 of our Ocatave.

So now let's calculate the Frequency-Ratio for the Pythagorean-Comma with 23,46 Cent:

$$\operatorname{Comma-Expected} = 2^{\frac{23,46\, Cent}{1200\, Cent}} \approx 1,0136$${#1:10141}

If we look at our Results for the Target-Frequencies of 12 Fiths ($1,5^{12}$) and 7 Octaves ($2^7$), our self-calculated Pythagorean-Comma can be determinded by:

$$\operatorname{Comma-Calculated} = \frac{1,5^{12}}{2^{7}} = \frac{129,75}{128,00} \approx 1,0136$${#2:10142}

Since [(1)](#10141) and [(2)](#10142) result in the same Value, my Calculation seems to be correct and even more Digits behind the [Comma](600007.md) look also quite promising:

![Pythagorean-Comma Comparison](400000037.jpg)

But I still don't have the Feeling to really have understood this [Music-System](90000001.md) completely, so probably it takes a little bit more Time until I can say "[q. e. d.](13000024.md)" with enough Confidence.

###### Topics: Hash-Functions

The last Days I have dealt with the Topic [Hash-Functions](13000019.md): In [Informatics](9000000.md) you find them everywhere, for Example [Git](9000197.md) uses [SHA-1](9000196.md) or [Bitcoin](130000023.md) utilizes [SHA-256](9000199.md), which is one Member of the [SHA-2-Family](2000198.md).

The Algorithms mentioned have been developed by the [NSA](141000005.md) and standardized by the [NIST](141000006.md). Both Institutions are located in the [USA](141000004.md).

Hash-Functions are used to reduce big Numbers to small Ones. A simple Secimen for an Hash-Algorithm is the [Digit-Sum](13000020.md), where for Example the Number 4711 is mapped to the Number 13.

A Problem of [Hash-Sums](13000019.md) are the so called [Hash-Collisions](2000200.md): Since the Input-Set is bigger than the Output-Set it is unavoidable that there are two Input-Numbers that map to the same Output-Number.

Here an Example for our Digit-Sum, where the Numbers 4711 and 1003 lead both to the Number 13:

```
Digit-Sum(4711) = 13 (4 + 7 + 1 + 1)
Digit-Sum(1003) = 13 (1 + 0 + 0 + 1)
```

One Quality of a Hash-Algorithm is its [Collision-Resistence](2000201.md), which is the Likelihood that this happens.

###### Topics: Information-Sharing, Internet, Xanadu, Virtual-Mapping

So, [after](#3201) thinking a little bit about Information-Sharing Theory, let's have a Look on the History of [Internet](9000152.md):

The British Man [Tim_Berners-Lee](70000014.md) invented [HTML](9300001.md) that provides [Hyper-Links](9300003.md) for Referencing Information in the [World-Wide-Web](9300004.md).

A Link can be specified as follows:

```html
<a href="https://holzheu.de/">Web-Site from Michael_Holzheu</a>
```

Like with Markdown, there seems to be no native Mechanism [[3211]](#3211) in HTML to include other HTML-Pages, but interestingly including Pictures is possible:

```html
<img src="0.jpg" alt="Michael_Holzheu"/>
```

The US Man [Ted_Nelson](70000015.md) suggested an Alterntive to the Ideas of [Tim_Berners-Lee](70000014.md) and implemented the [Xanadu-System](9300005.md), which is able to include Information in other Documents using so-called [Transclusion](404.md), where the Data is copied only at [Rendering](9000052.md)-Time.

The Concept of Transclusion is a very wide-spread Thing that is called Mapping in Virtualization, e.g. the following three Examples immediately come into my Mind:

- Mappings with the [MMap-Unix-System-Call](404.md)
- References in [Spread-Sheets](700038.md)
- Include-Files in [Programming-Languages](9010000.md)

Here a small Diagram that illustrates the Concept of Virtual-Mapping at the Example of two different Virtual-Documents that are constructed out of three "real" Objects:

```
+---------------------------------------------------------+
|                                                         |
|       Virtual-1          Real           Virtual-2       |
|                                                         |
| +-------------------+             +-------------------+ |
| |     Object-3      |  Object-1   |     Object-1      | |
| |                   |             |                   | |
| |     Object-2      |  Object-2   |     Object-2      | |
| |                   |             |                   | |
| |     Object-1      |  Object-3   |     Object-3      | |
| +-------------------+             +-------------------+ |
|                                                         |
+---------------------------------------------------------+
```

Advantages of Virtual-Mappings:

- Resource-Consumption: The References require less Ressources than the real Objects
- Consistency: When a real Object is changed, all virtual Objects automatically get the Update

Disadvantages of Virtual-Mappings:

- External-References: When mapping a real Object also all of its External-References have to be somewhere mapped in the Virtual-Document.
- Reference-Relocation: If References have a Local-Name they have to be renamed in the Virtual-Document, this Process is also Sometimes called Rellocation.
- Resource-Consumption: The Resolution of References in the Rendering-Process also takes some Time.
- Consistency: When one Physical-Object is updated and the Meaning changes, the overall Consisteny in the Virtual-Document still have to be ensured somehow.

```bash
function Update_Sub_Modules()
{
        local Tag=$1
        local Old_Tag;

        for Git_Sub_Module in $(git submodule | awk '{print $2}')
        do
                >&2 echo "Updating: $Git_Sub_Module"

                Old_Tag=$(git tag -l | grep $Tag)
                cd $Git_Sub_Module
                git pull origin Master
                git checkout Master
                git pull --rebase

                if [ "$Old_Tag" != "" ]; then
                        git tag -d $Tag
                fi
                git push origin :refs/tags/$Tag
                git tag $Tag
                git push --tags;
                cd ..
        done
}
```

Blog-Chain-Time 2023.04.12-17:23 in Tübingen, Yesterday I was thinking on the Problem, how to reference Information in my Documents.

References have two Elements, namely the Anchor that marks the Location of an Information and a Link from another Document that adressess that Anchor.

In Markdown there is the Concept of local Links that are automatically created [[1910]](1910) for each Heading. Unfotunately it looks like that there is no common [Standard](60047.md), how the Anchors are created.

Let's use the following Heading as Example:

Unfortuately the Git-Hub Markdown assumes a different Anochor for the Heading, namely `blog-chain-time-20230412-1446-in-tübingen`, and therefore the generated Link does not work.

I this Project for Addressing of Number-Files we already use its Number, so my new Idea is to also that Concept for [Addressing](600053.md) of Information within Documents. Like so often in Informatics you try around until it works, but back to the Concepts:

Currently we have three different Types of [Information-Containers](7000002.md):

- Repositories
- Number-Files
- Documents

Repositories contain Number-Files, Number-Files contain Documents, Documents contain e.g. Blog-Entries [[1911]](#1911). We have the following Addressing for every Piece of Information that we can use for [References](60039.md) from other Information:

- Repository-Number
- Number-File-Number
- Html-Tag-Number

The first two Numbers currently are unique in the Scope of this Project, but the Html-Tag-Number is only unequivocal within the Document.

What makes the Things even more complicated is the Fact that Information Containers can change over Time. So when we e.g. reference a Blog-Entry from another Blog-Entry, after some Days the Content of the first one could have canged, which then makes the Reference from the second one somehow invalid.

There are several Solutions to that Problem:

1. Use a Time-Stamp in the Reference
2. Use a Hash-Value for the Information in the Reference
3. Don't use References and instead copy the Information
4. Try to keep the whole Project consistent at one Point in Time

My current Approach to solve the Problem of [Consistency](600054.md) is Solution 4. Therefore I try make one Day as consistent as possible. With Git it is still possible to checkout and render old Days. So if you are interested in the [Truth](600021.md) of the [Past](10000019.md) you still can find it.

And if some Information changes or moves, I try my best to update all the References in the Project so that the current Day is halfway consistent again.

```console
17:17 $ cat 300000004.make
# # All-Things-Makefile-File
#

Md-File-List = $(wildcard *.md)
Html-File-List = $(subst .md,.html,$(Md-File-List))

all: Html-Files
	echo "All-Things-Makefile-File"

Html-Files: $(Html-File-List)

%.html: %.md
	pandoc --standalone --template 300000002.htm $< -o $@
	sed -i '' -E 's/(href="[0-9]+)\.md/\1\.html/g' $@

clean:
	rm -f $(Html-File-List)

.PHONY: all clean
```
